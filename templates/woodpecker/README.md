# Woodpecker self-hosted CI scaffold

Install templates for a self-hosted [Woodpecker](https://woodpecker-ci.org) CI
server + agent, generated by `/cicd:woodpecker`. These are the "install / secrets
bootstrap" half of the Woodpecker skill; the pipeline itself is generated
separately into `.woodpecker.yml`.

Full patterns and gotchas: `docs/skills/woodpecker-ci.md`.

## Files

| File | Purpose |
|------|---------|
| `docker-compose.server.yml.example` | Woodpecker server (UI/API + agent gRPC) |
| `docker-compose.agent.yml.example` | Woodpecker agent (runs steps as containers) |
| `bootstrap-secrets.py.example` | Fetch forge/agent secrets from AWS Secrets Manager into `docker.env` |

## Quick start

```bash
# 1. Copy the templates into a woodpecker/ dir (drop the .example suffix)
mkdir -p woodpecker && cd woodpecker
cp /path/to/templates/woodpecker/docker-compose.server.yml.example docker-compose.server.yml
cp /path/to/templates/woodpecker/docker-compose.agent.yml.example  docker-compose.agent.yml
cp /path/to/templates/woodpecker/bootstrap-secrets.py.example      bootstrap-secrets.py

# 2. Bootstrap server secrets from AWS Secrets Manager (writes gitignored docker.env)
python3 bootstrap-secrets.py --secret-name my-woodpecker-secret --region us-east-1

# 3. Set WOODPECKER_ADMIN in docker-compose.server.yml to your forge username, then:
docker compose -f docker-compose.server.yml up -d

# 4. On each executor host, create agent.env (WOODPECKER_SERVER + WOODPECKER_AGENT_SECRET):
docker compose -f docker-compose.agent.yml up -d
```

## Notes

- `docker.env` and `agent.env` hold secrets - add them to `.gitignore`.
- Keep the agent gRPC port (`:9001`) private: Tailscale or LAN only, never public.
- Pin image digests (`tag@sha256:...`) and rotate them with Renovate. The
  examples ship tag-only for readability; add digests before production use.
- The forge OAuth app (GitHub client id + secret) and the shared
  `WOODPECKER_AGENT_SECRET` are the credentials `bootstrap-secrets.py` expects in
  the AWS secret JSON.
